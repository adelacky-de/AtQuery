# Skill: Verification (Antigravity Assistant)

Guides me through proving that my changes work as intended.

## Process
1. **Logic Tracing**: Trace the data flow of the new code.
2. **File Audit**: Check for dangling imports or unused variables.
3. **Telemetry**: Ensure "HINT FOR AI" and error messages are descriptive.
4. **User Walkthrough**: Present the changes clearly and summarize the testing.

## Anti-Rationalizations
| Agent Excuse | Rebuttal |
| :--- | :--- |
| "The code looks good, it must work." | **NO.** "Looks good" is not a proof. Run checks. |
| "I've already tested this in my head." | **NO.** Physical file audits and dry-runs are mandatory. |

## Verification Gates
- **Functional Self-Check**: Does the change actually solve the user's problem?
- **Zero Regressions**: Confirm that existing functional parts are still working.
