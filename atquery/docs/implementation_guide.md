# AtQuery: Step-by-Step Implementation Guide

This guide describes how to build the AtQuery plugin from scratch, starting from a fresh QGIS installation.

## Phase 1: Environment Setup

1.  **Install QGIS**: Download and install the latest LTR (Long Term Release) of QGIS from [qgis.org](https://qgis.org).
2.  **Install Ollama**: Download and install Ollama from [ollama.com](https://ollama.com). 
    *   Once installed, open your terminal and run: `ollama pull llama3.1` (or your preferred model).
3.  **Python Environment**: QGIS comes with its own Python environment. Ensure you have `requests` installed. In the QGIS Python Console, you can check with:
    ```python
    import requests
    ```

## Phase 2: Create the Plugin Skeleton

1.  **Open QGIS**.
2.  **Install Plugin Builder**:
    *   Go to **Plugins** -> **Manage and Install Plugins...**
    *   Search for **"Plugin Builder 3"**.
    *   Click **Install Plugin**.
3.  **Generate Plugin**:
    *   Open the Plugin Builder tool (typically found in the Plugins menu or toolbar).
    *   Fill in the mandatory fields:
        *   **Class Name**: `AtQuery`
        *   **Plugin Name**: `AtQuery`
        *   **Module Name**: `atquery`
        *   **Description**: `AI-Powered Natural Language Interface for QGIS`
    *   **Configuration**: When prompted, ensure the option to **"Include pb_tool configuration"** is checked. This generates your `pb_tool.cfg` file.
    *   Choose **"Dock Widget"** as the template.
    *   Select a folder on your computer to save the generated code.

## Phase 3: Implement the Logic

1.  **Core Thinking (`core/ai_brain.py`)**:
    *   Define the tool schemas that the AI will use (e.g., `get_layer_list`, `select_features`).
    *   Write the system prompt that instructs the AI on how to behave as a GIS agent.
2.  **User Interface (`ui/AtQuery_dockwidget.py`)**:
    *   Design the chat window with a `QTextEdit` for history and a `QLineEdit` for user input.
    *   Implement the asynchronous loop that sends user text to Ollama and handles resulting tool calls.
3.  **Setup Helpers (`utils/installer_utils.py`)**:
    *   **Ollama Connectivity**: Write logic to ping `http://localhost:11434/api/tags` to check if the Ollama server is reachable.
    *   **User Warning System**: Implement a check that triggers a warning dialog if Ollama is not installed or the service is not running. This prevents the plugin from hanging when the AI tries to call the local model.

## Phase 4: Project Organization

To keep the project clean, move your files into subdirectories as established in this project:
*   `core/` for AI logic.
*   `ui/` for widgets.
*   `resources/` for icons.
*   `config/` for build files.

**Important**: Update your imports (e.g., `from .ui.AtQuery_dockwidget import AtQueryDockWidget` in `AtQuery.py`) to match the new folder structure.

## Phase 5: Build and Deploy

1.  **Compile Resources**:
    *   QGIS plugins use a `.qrc` file (like `resources.qrc`) as an XML manifest of icons and assets. These must be "compiled" into Python code so they can be imported as modules.
    *   **The Generated Files**: This process produces files like `resources.py` and `resources_rc.py`. 
        > [!NOTE]
        > These files contain large blocks of binary data converted to strings. They are **not intended for manual editing** and are completely machine-generated.
    *   **Using `pb_tool` (Recommended)**: 
        *   `pb_tool` is a specialized CLI for QGIS developers. 
        *   **Action**: Running `pb_tool deploy` or `pb_tool compile` automatically reads your `.qrc` and generates these Python files.
    *   **Using `pyrcc5` (Manual)**:
        *   **Action**: Run `pyrcc5 -o resources.py resources.qrc`. 
        *   If your plugin expects a different filename (like `resources_rc.py`), you would run `pyrcc5 -o resources_rc.py resources.qrc`.
2.  **Load in QGIS**:
    *   Ensure your plugin folder is in the QGIS plugins directory (e.g., `~/Library/Application Support/QGIS/QGIS3/profiles/default/python/plugins`).
    *   Restart QGIS or use the **"Plugin Reloader"** plugin to load AtQuery.
3.  **Start Chatting**: Click the AtQuery icon in the toolbar and start typing!
